# IM 정책

## 아이템 관리 정책

### 추가 방식 (Additive)

스캔/구조화 시 기존 아이템을 삭제하지 않고 **새 아이템만 추가**한다.

- `appendItems()` — 기존 아이템 유지 + 새 아이템 추가
- `replaceItems()` — 브레인스토밍 재구조화 시에만 사용 (unpinned 아이템 교체)

### 핀(고정) 정책

| 상태 | 동작 |
|------|------|
| 핀 고정 (is_pinned=1) | 재구조화 시에도 삭제되지 않음 |
| 핀 해제 (is_pinned=0) | 브레인스토밍 재구조화 시 교체 대상 |

### 잠금 정책

| 상태 | 동작 |
|------|------|
| 잠금 (is_locked=1) | MCP에서 접근 불가. 기본값 |
| 해제 (is_locked=0) | MCP `get-next-task`에 노출됨 |

- 상위 항목 잠금 시 하위 전체 자동 잠금
- 완료(done) 상태 전환 시 자동 잠금

### 삭제 정책

- **개별 삭제**: 항목 hover → 삭제 버튼 (자식 포함 재귀 삭제)
- **선택 삭제**: 선택 모드 → 체크박스 → 선택삭제 버튼
- **전체 삭제**: 전체삭제 버튼 (프로젝트의 모든 아이템 삭제)

## 프로젝트 스캔 정책

### 스캔 범위

- 재귀 탐색 (전체 디렉토리)
- 디렉토리 트리는 depth 3까지만
- 무시 디렉토리: node_modules, .git, .next, dist, build 등

### 파일 우선순위

1. 프로젝트 설정 (package.json, README.md, CLAUDE.md 등)
2. 문서 (.md, .txt, .rst)
3. 엔트리 포인트 (index.*, main.*, app.*)
4. 설정 파일 (.json, .yaml, .toml)
5. 소스코드 (.ts, .tsx, .js, .py, .go 등)

### 크기 제한

| 항목 | 제한 |
|------|------|
| 파일당 | 100KB |
| 총합 | 1MB |
| AI 컨텍스트 단일 호출 | 100KB |
| AI 청크당 | 80KB |

## AI 정책

### 엔진

Claude Code CLI를 직접 spawn. `@anthropic-ai/claude-agent-sdk`는 사용하지 않음.

### 모델

기본 `sonnet`. `--max-turns 1`로 단일 턴 응답.

### 출력 형식

모든 AI 응답은 JSON만 허용. 마크다운 펜스가 있으면 자동 제거 후 파싱.

### 대규모 프로젝트

100KB 초과 시 서브 프로젝트별 분할 분석. 50KB 미만 소규모는 병합하여 AI 호출 횟수 최적화.

## 데이터 정책

- 모든 데이터는 로컬 (`~/.idea-manager/data/im.db`)
- 클라우드 전송 없음
- API 키 불필요 (사용자 Claude 구독 사용)
