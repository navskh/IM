# IM v2 — Claude Code 워크플로우 매니저

## 타겟
- Claude Code CLI 사용자 (구독자)
- 여러 프로젝트를 바이브 코딩으로 진행하는 개발자

## 배포
- npm 패키지 (`im start` → 브라우저 오픈)
- 데이터: 로컬 SQLite (~/.idea-manager/)

## 핵심 철학
- **AI 없이도 완벽히 동작** — 조직화 도구가 핵심
- **AI는 선택적 보조** — 사용자가 명시적으로 요청할 때만 CLI 호출
- **프롬프트가 최종 산출물** — 아이디어 → 태스크 → 프롬프트 정제 과정 관리

## 계층 구조

```
프로젝트 (JABIS, ARA)
  └─ 서브 프로젝트 (jabis-teamlead, jabis-cert...)  ← 카드
      └─ 태스크 (근태 신청 폼, OAuth 연동...)
          ├─ 메모 (브레인스토밍에서 온 내용)
          ├─ AI 대화 (선택적, 구체화 과정)
          └─ 프롬프트 (최종 산출물, 복사해서 사용)
```

## 화면 구성

### 1. 대시보드
- **탭 3개**: [🔥 진행 중] [📋 전체] [⭐ 오늘 할 일]
  - 진행 중: 전체 프로젝트에서 활성 태스크가 있는 서브 프로젝트만
  - 전체: 프로젝트 > 서브 프로젝트 전부
  - 오늘 할 일: 사용자가 직접 태깅한 태스크
- **서브 프로젝트 카드**:
  - 할 일 제목 미리보기 (해야하는 것 중심)
  - 상태 요약 (진행 중 N개 / 대기 N개)
  - 마지막 작업 시간
  - 진행 중인 카드가 먼저 정렬

### 2. 브레인스토밍 (프로젝트 레벨)
- 프로젝트 전체에 대한 자유 형식 메모 영역
- 텍스트 선택 → 단축키(Cmd+T)로 태스크 생성
- 태스크 생성 시 서브 프로젝트 배정 (없으면 자동 생성)

### 3. 태스크 상세
- **상태 흐름**: 💡 아이디어 → ✏️ 작성 중 → 🚀 투입 → 🧪 테스트 → ✅ 완료 / 🔴 문제
- 단축키로 상태 변경 (Cmd/Ctrl + 1~5)
- 클릭으로도 상태 변경
- **프롬프트 에디터**: 최종 산출물, 복사 버튼
- **AI 대화** (선택적): 태스크 구체화 논의
- **AI 인라인 제안** (선택적): 프롬프트 쓸 때 보조 코멘트

## 서브 프로젝트 생성 방식 (A+B+C)
- **A. 디렉토리 자동 감지**: 프로젝트 폴더 연결 시 하위 디렉토리 스캔 → 서브 프로젝트 자동 생성
- **B. 수동 생성**: "+ 서브 프로젝트 추가" 버튼 → 이름 입력, 폴더 연결은 선택
- **C. 태스크 배정 시 자동 생성**: 없는 서브 프로젝트명 입력하면 자동 생성
- 서브 프로젝트는 실제 폴더와 1:1 매칭이 아닐 수 있음 (개념적 그룹도 가능)

## AI 보조 (선택적)
- Claude Code CLI 활용 (구독자 대상)
- 사용자가 명시적으로 요청할 때만 호출
- 기능:
  - "프롬프트 다듬기" 버튼
  - AI 채팅 (태스크 구체화)
  - 인라인 제안 (프롬프트 작성 시)
  - 프로젝트 디렉토리 컨텍스트 참조
- 실패해도 도구 자체는 정상 동작

## 단축키 (Cmd/Ctrl 기반, Mac/Windows 호환)
- `Cmd+T`: 브레인스토밍 텍스트 → 태스크 생성
- `Cmd+1~5`: 태스크 상태 변경 (아이디어/작성중/투입/테스트/완료)
- `Cmd+C` (프롬프트 영역): 프롬프트 복사
- `Cmd+Enter`: AI에게 질문

## 기술 스택

- SQLite (better-sqlite3, 로컬 데이터)
- Claude Code CLI (선택적 AI 보조)
