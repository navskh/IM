# Idea Manager (IM) - 기능 정의서

## 화면 구조

### 메인 화면: 2패널 + 하단 AI 대화

```
┌─────────────────────────┬─────────────────────────┐
│                         │                         │
│  📝 브레인스토밍 패널   │  🗺️ 구조화 뷰 패널     │
│                         │                         │
│  자유 입력 영역         │  트리 구조 (정리된 결과) │
│  + 📌 AI 메모 (인라인) │  항목 클릭 → 상세보기   │
│                         │                         │
├─────────────────────────┤                         │
│                         │                         │
│  💬 AI 대화 영역        │                         │
│  (하단 채팅)            │                         │
│                         │                         │
└─────────────────────────┴─────────────────────────┘
```

---

## 기능 1: 브레인스토밍 (왼쪽 상단)

### 사용자가 할 수 있는 것
- 자유롭게 아이디어를 텍스트로 입력
- 마크다운 지원 (제목, 리스트, 볼드 등)
- 입력 중간에 AI 메모가 인라인으로 붙음 (📌)

### 동작 방식
- 텍스트 입력은 자동 저장 (디바운스)
- 타이핑 멈추면 (3초) AI가 자동으로 분석 시작
- AI가 애매하거나 빠진 부분을 발견하면:
  - 해당 텍스트 옆에 📌 메모로 질문 표시 (인라인)
  - 동시에 하단 AI 대화 영역에도 질문 표시

### 화면 예시
```
"근태 관리 만들어야 하는데
 출퇴근은 GPS로 하고 싶고"     📌 반경 밖이면?

"반경 200m 안에서만 되게 하고   📌 200m 고정?
 근태 수정도 필요한데 상급자
 결재가 있어야 하고..."
```

## 기능 2: AI 대화 (왼쪽 하단)

### 사용자가 할 수 있는 것
- AI 질문에 답변 (텍스트 입력)
- AI에게 추가 질문 가능
- 메모(📌) 클릭 시 해당 대화로 이동

### 동작 방식
- AI가 브레인스토밍 내용을 분석 후 질문을 생성
- 사용자가 답하면 → 구조화 뷰에 자동 반영
- 대화 이력은 저장됨 (나중에 다시 볼 수 있음)

### 화면 예시
```
─── AI 대화 ───

🤖 3가지 확인할게요:
  1. 반경 밖 출근 시도 → 어떻게 처리?
  2. 200m는 고정? 설정 가능하게?
  3. 퇴근도 GPS 체크 필요?

나: 1번은 에러 메시지, 2번은 고정, 3번은 필요없어

🤖 반영했어요. 구조화 뷰를 업데이트했습니다.
   추가로, 근태 수정의 결재 프로세스는
   어떤 흐름을 생각하고 있나요?
```

## 기능 3: 구조화 뷰 (오른쪽)

### 사용자가 할 수 있는 것
- AI가 정리한 트리 구조를 확인
- 각 항목 클릭 → 상세보기 (설명 + 프롬프트)
- 항목별 잠금/해제 설정
- 항목 상태 변경 (대기 → 진행중 → 완료)
- 항목 수동 추가/수정/삭제/순서변경

### 동작 방식
- 브레인스토밍 내용이 변경되면 AI가 자동으로 트리 업데이트
- 기존 항목은 유지하면서 새로운 내용만 추가 (증분 업데이트)
- 사용자가 수동으로 수정한 항목은 AI가 덮어쓰지 않음

### 트리 뷰 화면 예시
```
📋 근태 관리 시스템
├── 🔓 GPS 출퇴근 기록        ✅ 완료
├── 🔓 출근 확인 UI           🔄 진행중
├── 🔐 근태 수정/결재         ⏳ 대기
│   ├── 🔐 결재자 관리
│   ├── 🔐 결재 프로세스
│   └── 🔐 알림 기능
└── 🔐 반응형 대응            ⏳ 대기
```

### 상세보기 화면 예시 (항목 클릭 시)
```
┌───────────────────────────┐
│ 📄 GPS 출퇴근 기록        │
│                           │
│ 설명:                     │
│ GPS API를 연동하여 현재   │
│ 위치 기반 출퇴근 기록.    │
│ 반경 200m(고정) 내에서만  │
│ 출근 가능. 범위 밖이면    │
│ 에러 메시지 표시.         │
│                           │
│ 상태: ✅ 완료             │
│ 우선순위: 🔴 높음         │
│ 잠금: 🔓 해제             │
│                           │
│ ── 프롬프트 ──            │
│ "GPS API를 연동해서       │
│  현재 위치 기반으로       │
│  출퇴근을 기록하는        │
│  기능을 만들어줘.         │
│  조건:                    │
│  - 반경 200m 내 출근 가능 │
│  - 범위 밖이면 에러 메시지│
│  - 퇴근은 GPS 체크 안 함" │
│                           │
│ [📋 복사] [🔓 실행 허용]  │
└───────────────────────────┘
```

## 기능 4: 프롬프트 생성

### 사용자가 할 수 있는 것
- 구조화된 항목의 상세보기에서 프롬프트 확인
- 프롬프트 복사 (클립보드)
- 프롬프트 재생성 요청
- 프롬프트 직접 수정

### 동작 방식
- 구조화 시 각 항목에 대한 프롬프트를 AI가 자동 생성
- 프롬프트는 바이브 코딩 도구에 바로 넣을 수 있는 형태
- 브레인스토밍 + AI 대화 내용을 모두 반영하여 생성
- 사용자가 수정하면 수정본이 우선

### 프롬프트 생성 규칙
- 구체적이고 실행 가능한 지시문
- 조건/제약사항 포함
- 관련 컨텍스트 (이전 작업, 의존성) 포함
- 기술적 세부사항은 사용자 프로젝트에 맞게

## 기능 5: 실행 범위 제어 (잠금/해제)

### 사용자가 할 수 있는 것
- 각 항목의 잠금(🔐) / 해제(🔓) 토글
- 상위 항목 잠금 시 하위도 자동 잠금
- 해제된 항목만 MCP를 통해 외부 도구에 노출

### 동작 방식
- 신규 항목은 기본 잠금(🔐) 상태
- 사용자가 검토 후 직접 해제
- MCP 연동 시 해제된 항목의 프롬프트만 제공
- 완료된 항목은 자동으로 다시 잠금

### 상태 흐름
```
[신규 생성] → 🔐 잠금 (대기)
    ↓ 사용자가 해제
         🔓 해제 (실행 가능)
    ↓ MCP로 실행 도구가 가져감
         🔄 진행중
    ↓ 완료 보고
         ✅ 완료 → 🔐 자동 잠금
```

## 기능 6: MCP Server 연동

### 외부 도구에서 할 수 있는 것
- 다음 할 일 조회 (해제 + 대기 상태인 항목)
- 프로젝트 전체 컨텍스트 조회
- 특정 항목의 프롬프트 조회
- 작업 완료 보고
- 작업 상태 변경

### 연동 방법
```bash
# CLI에서 MCP 서버 모드 실행
im mcp

# Claude Code 설정에 추가
# claude_desktop_config.json
{
  "mcpServers": {
    "idea-manager": {
      "command": "im",
      "args": ["mcp"]
    }
  }
}
```

### MCP 도구 목록

| 도구명 | 설명 |
|--------|------|
| `get-next-task` | 다음 실행 가능한 작업 + 프롬프트 조회 |
| `get-project-context` | 프로젝트 전체 구조/상태 조회 |
| `get-prompt` | 특정 항목의 프롬프트 조회 |
| `report-completion` | 작업 완료 보고 |
| `update-status` | 작업 상태 변경 |
| `list-projects` | 프로젝트 목록 조회 |

## 전체 사용자 플로우

```
1. im start → 브라우저 열림
      ↓
2. 프로젝트 생성 (또는 기존 프로젝트 선택)
      ↓
3. 왼쪽 패널에 아이디어 자유롭게 입력
      ↓
4. (3초 후) AI가 자동으로:
   - 오른쪽에 구조화 트리 생성
   - 왼쪽에 📌 메모 + 하단 💬 질문
      ↓
5. 사용자가 AI 질문에 답변 → 구조화 뷰 자동 업데이트
      ↓
6. 구조화 뷰에서 항목 확인 → 상세보기로 프롬프트 확인
      ↓
7. 실행할 항목 잠금 해제 (🔐 → 🔓)
      ↓
8. 방법 A: 프롬프트 복사 → Cursor/Claude Code에 붙여넣기
   방법 B: MCP 연동 → 실행 도구가 자동으로 프롬프트 가져감
      ↓
9. 완료 보고 (수동 또는 MCP 자동) → 상태 업데이트
      ↓
10. 다음 항목 확인 → 7번으로 반복
```
