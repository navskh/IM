# Idea Manager (IM)

> **"I'm your Idea Manager"**

## 한 줄 요약
바이브 코딩을 위한 아이디어 관리 & 프롬프트 생성 플랫폼

## 문제 정의
바이브 코딩은 아이디어를 실체화하는 데 빠르지만, 그 전 단계인 **아이디어 정리**에 시간이 많이 걸린다.

현재 워크플로우 (노션 기반):
1. 노션에 할 일을 자유롭게 적음
2. 그걸 프롬프트로 수작업 변환
3. 자세하게 하기 위해 넣고 빼고 반복
4. 바이브 코딩 도구에서 실행
5. 어디까지 진행했는지 체크, 테스팅
6. 다음 스텝 생각 → 다시 프롬프트 만들기
7. 반복...

### 핵심 불편 포인트
- 생각을 적는 곳과 프롬프트를 만드는 곳이 분리되어 있음
- 진행상태 추적이 수동 (취소선 + 태그)
- 할 일, SQL, 메모, 아이디어가 뒤죽박죽 섞임
- "다음 뭐하지?"를 매번 스스로 판단해야 함
- 같은 내용을 두 번 씀 (노션 + 프롬프트)

## 핵심 가치
> "생각을 쏟아내기만 하면, 나머지는 IM이 해준다"

## 타겟 사용자
바이브 코딩을 하는 모든 사람

## 포지셔닝
```
노션 (자유롭지만 다 수동)
   ↕ IM이 여기
Jira (체계적이지만 딱딱함)
```
- 노션처럼 자유도가 높되
- AI가 구조화 + 프롬프트 생성까지 해주는 서비스

## 핵심 기능

### 1. 실시간 2패널 UI

왼쪽: 브레인스토밍 (자유 입력)
오른쪽: AI가 실시간으로 구조화한 뷰 (트리 형태)

```
┌──────────────────────┬──────────────────────┐
│                      │                      │
│  💬 브레인스토밍     │  🗺️ 구조화 뷰       │
│                      │                      │
│  자유롭게 적기       │  AI가 실시간 정리    │
│  아이디어 쏟아내기   │  트리 구조로 시각화  │
│                      │  상태 표시           │
│                      │  [프롬프트 생성]     │
│                      │                      │
└──────────────────────┴──────────────────────┘
```

### 2. AI 자동 구조화
- 브레인스토밍 내용을 실시간으로 분석
- 기능 분류, 우선순위, 의존관계 파악
- 트리 구조로 정리하여 시각화

### 3. 프롬프트 자동 생성
- 구조화된 작업 항목에서 바이브 코딩용 프롬프트 생성
- 바로 실행 도구에 넣을 수 있는 형태로 출력

### 4. 실행 범위 제어 (잠금/해제)
- 사용자가 각 작업 항목의 실행 허용 여부를 설정
- 잠금: AI가 가져갈 수 없음 (아직 설계 안 됨, 검토 필요 등)
- 해제: AI 실행 도구가 MCP를 통해 가져갈 수 있음

```
📋 근태 관리 시스템
├── 🔓 1. GPS 출퇴근 기록        → 실행 허용
├── 🔓 2. 출근 확인 UI           → 실행 허용
├── 🔐 3. 근태 결재자 관리       → 잠금
├── 🔐 4. 알림 기능              → 잠금
└── 🔐 5. 반응형 대응            → 잠금
```

### 5. MCP Server 연동
IM이 MCP Server로 동작하여 외부 실행 도구와 연동

```
IM (MCP Server)
  ↕ MCP 프로토콜
Cursor / Claude Code / Windsurf 등 (MCP Client)
```

MCP를 통해 가능한 것:
- 다음 할 일 (허용된 프롬프트) 조회
- 완료 보고 → IM에 상태 자동 업데이트
- 프로젝트 컨텍스트 전달
- 이전 작업 히스토리 조회

### 6. 프로젝트 관리
- 프로젝트 단위로 아이디어/프롬프트 관리
- 전체 설계도 (트리 뷰)로 진행상황 한눈에 파악
- 진행상태 자동 추적

---

## 전체 워크플로우

```
1. [사용자] 브레인스토밍 자유 입력
      ↓
2. [IM AI] 실시간 구조화 (트리 뷰)
      ↓
3. [사용자] 확인 후 실행 범위 설정 (잠금/해제)
      ↓
4. [IM AI] 해제된 항목의 프롬프트 생성
      ↓
5. [MCP] 실행 도구가 프롬프트 가져감
      ↓
6. [MCP] 실행 완료 보고 → 상태 자동 업데이트
      ↓
7. [사용자] 다음 단계 확인 → 잠금 해제 → 반복
```

## 아키텍처

### AI 엔진: Claude Code SDK
IM은 자체 AI API를 사용하지 않는다. 대신 **Claude Code SDK** (`@anthropic-ai/claude-code`)를 AI 엔진으로 사용한다.

- API 키 관리 / 별도 과금 없음 (사용자의 Claude 구독으로 동작)
- Claude Code의 모든 기능 (파일 읽기, 코드 생성 등) 그대로 활용

```typescript
import { claude } from '@anthropic-ai/claude-code';

// IM 내부에서 Claude Code를 AI 엔진으로 호출
const result = await claude({
  prompt: "이 브레인스토밍 내용을 구조화해줘: ...",
  options: { maxTurns: 3 }
});
```

### 배포 방식: npm 패키지 (로컬 실행)

```bash
# 설치
npm install -g idea-manager

# 실행
im start
```

실행하면 로컬에서 다음이 동시에 시작:

```
사용자 PC (localhost)
├── Next.js 웹앱 (localhost:3000) ← 브라우저에서 접속
├── Claude Code SDK              ← AI 처리 (로컬)
├── MCP Server                   ← Cursor/Claude Code 연동 (로컬)
└── 로컬 데이터 저장 (JSON/SQLite)
```

### 이 방식의 장점
- **설치 한 줄, 실행 한 줄** (`im start`)
- **클라우드 서버 비용 없음** (모든 것이 로컬)
- **사용자 데이터가 로컬에 저장** (프라이버시 보장)
- **Claude Code 인증 그대로 사용** (별도 API 키 불필요)
- **MCP Server도 로컬**이라 바로 연결 가능

### 실행 시 화면

```bash
$ im start

  IM - Idea Manager v1.0.0

  🌐 Web UI:    http://localhost:3000
  🔌 MCP Server: localhost:3100
  📁 Data:       ~/.idea-manager/data/

  Ready! Opening browser...
```

## 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| Frontend | Next.js + React + TypeScript | 웹 UI |
| AI 엔진 | Claude Code SDK (`@anthropic-ai/claude-code`) | API 대신 로컬 SDK |
| 데이터 저장 | SQLite (또는 JSON) | 로컬 파일 기반 |
| MCP Server | TypeScript MCP SDK | 외부 도구 연동 |
| 배포 | npm 패키지 | `npm install -g idea-manager` |
| CLI | Commander.js (또는 유사) | `im start`, `im init` 등 |
